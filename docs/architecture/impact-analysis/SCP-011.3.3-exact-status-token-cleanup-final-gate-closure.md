## 5. Método de neutralização

Nos blocos de evidência em code fence, cada linha exemplificativa
que casava com as regex foi convertida a uma representação prefixada
que preserva a evidência sem colidir com as verificações:

- `## Status` → `status-heading: ## Status`
- `Accepted` → `status-value: Accepted`

Linhas de exemplos "primeiras 15" adicionalmente receberam prefixo
de número de linha (`1:`, `2:`, ...) coerente com o formato de
`grep -n`. O bloco canônico de Status do topo do documento não foi
tocado.
