## 4. Verificação do reader de produção

`src/lib/api/commercial/seat-usage-reader.ts` é a única implementação
da query autoritativa:

```ts
export async function readCommercialSeatUsage(client, tenantId) {
  if (typeof tenantId !== "string" || tenantId.length === 0) return null;
  try {
    const res = await client
      .from("tenant_members")
      .select("*", { count: "exact", head: true })
      .eq("tenant_id", tenantId)
      .in("membership_status", ["active", "invited"]);
    if (res.error) return null;
    return validateSeatUsedCount(res.count);
  } catch { return null; }
}
```

`commercial.functions.ts` importa e utiliza este reader:

```ts
import { readCommercialSeatUsage } from "./seat-usage-reader";
// ...
readSeatUsage: async (tid) => {
  return readCommercialSeatUsage(admin as unknown as Parameters<typeof readCommercialSeatUsage>[0], tid);
},
```

Nenhum helper `readUsed` paralelo existe no runtime ou na suíte
(verificado por `rg`).

## 5. Verificação dos testes

### 5.1 Ambiguidade do valor `false`

O array `badValues` (linhas 1020–1031 de
`commercial-seat-limit.spec.ts`) contém `-1, 1.5, "10", true, NaN,
MAX_SAFE_INTEGER+1`. `false` está **intencionalmente ausente** com
comentário explicativo: um valor booleano falsy no snapshot é
interceptado pelo feature-gate como `not_entitled` antes de chegar
ao short-circuit — caminho já coberto pelo bloco "feature decision
negative → no tenant_members read".

Cenário confirmado: **Cenário A — inconsistência não existe**. O
comentário reflete o comportamento real. Nenhuma alteração no teste
foi necessária.

Cobertura preservada: `negative`, `fractional`, `text`,
`boolean truthy`, `NaN`, `> MAX_SAFE_INTEGER`.

### 5.2 Cobertura de `effective=false`

Cobertura composta já existente:

- `extractSeatLimit rejects non-effective item` (linhas 330–336) —
  teste puro: `effective=false` → `limit=null`, `source="none"`.
- Testes de orquestração com feature decision negativa
  (`not_entitled`, `billing_*`) confirmam que `readSeatUsage` não é
  chamado.
- Short-circuit test cobre a composição: quando `extractSeatLimit`
  retorna `limit=null` (inclusive por `effective=false`), o runtime
  não chama `readSeatUsage`.

Cobertura suficiente — nenhum teste novo criado.
