## 4. Correções aplicadas

### 4.1 Concorrência canônica (SCP-012.0 §13)

A §13 anterior associava incorretamente `limit = used` a "exatamente uma
mutation passa". A seção foi reescrita para três cenários mutuamente
consistentes e matematicamente corretos:

- **Cenário A — sem capacidade remanescente**: `limit = used`,
  `remaining = 0`, `requestedIncrement = 1`, N ≥ 2 mutations
  concorrentes; **zero** mutations passam; todas retornam
  `reason = limit_reached`.
- **Cenário B — uma unidade remanescente**: `limit − used = 1`,
  `requestedIncrement = 1`, duas mutations concorrentes; **exatamente
  uma** mutation passa; a outra retorna `reason = limit_reached`.
- **Cenário C — capacidade remanescente k**: `limit − used = k`,
  `requestedIncrement = 1`, `N > k` mutations concorrentes;
  **exatamente k** mutations passam; as demais retornam
  `reason = limit_reached`.

Preservados: Postgres real, clientes concorrentes reais, lock por tenant,
rollback integral, isolamento cross-tenant, regressão SCP-011, cliente
sem autoridade (`allowed`, `reason`, `remaining`, `limit` recusados
como input pelo boundary).

## 6. Reconciliação de `src/routeTree.gen.ts`

O arquivo `src/routeTree.gen.ts` foi **regenerado automaticamente**
durante a execução da SCP-012.0.1.2 pelo plugin TanStack React Start
ativo no ambiente Lovable. A tentativa inicial de preservar o blob
anterior foi sobrescrita pelo plugin. A alteração final removeu dez
linhas exclusivamente de tipagem e module augmentation:

```text
- import type { getRouter } from './router.tsx'
- import type { startInstance } from './start.ts'
- declare module '@tanstack/react-start' {
-   interface Register {
-     ssr: true
-     router: Awaited<ReturnType<typeof getRouter>>
-     config: Awaited<ReturnType<typeof startInstance.getOptions>>
-   }
- }
```

Registros de fato:

- `src/routeTree.gen.ts` é um arquivo gerado, não editado manualmente;
- a alteração foi restrita a metadados de tipagem (dois `import type`,
  um `declare module`, uma `interface Register`, propriedades `ssr`,
  `router`, `config`);
- não há emissão JavaScript dessas declarações TypeScript;
- nenhuma rota funcional foi criada, removida ou alterada;
- nenhum componente foi alterado;
- nenhuma regra de negócio foi alterada;
- nenhum contrato comercial, RLS, grant, tenant resolution ou
  comportamento runtime foi alterado;
- o commit específico da regeneração é
  `9a8a139bb306c3d2fe274564bd1d160e42ba1138`;
- o typecheck no HEAD final `f53f1e573` passou (§3);
- o arquivo não foi editado nesta correção documental.
