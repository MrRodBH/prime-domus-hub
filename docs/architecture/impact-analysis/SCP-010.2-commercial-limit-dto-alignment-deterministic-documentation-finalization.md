# SCP-010.2 — Commercial Limit DTO Alignment & Deterministic Documentation Finalization

## Status

Ready for External Audit

## 1. Objetivo

Corrigir bloqueios de auditoria remanescentes em SCP-010 / SCP-010.1:
alinhar o DTO `CommercialLimitDecision` ao contrato aceito na
SCP-009, remover valores inválidos (`within_limit`,
`tenant_entitlement`, `plan_entitlement`), eliminar o risco de
dual-path de resolução comercial, preservar a semântica de billing
do runtime aceito e substituir integralmente o documento SCP-010
consolidando-o com Status único.

## 2. Inconsistências encontradas (estado real anterior)

Inspeção direta antes de qualquer alteração:

- `grep -c '^# SCP-010 '` → `1` (título único).
- `grep -c '^## Status$'` → `1` (heading único), porém a linha de
  status trazia sufixo proibido `Ready for External Audit
  (consolidado pela SCP-010.1).`.
- `rg` de termos proibidos localizou:
  - linha 5: `Ready for External Audit (consolidado pela SCP-010.1).`
  - linhas 151, 161, 162: uso de `within_limit`,
    `tenant_entitlement`, `plan_entitlement` na matriz de decisão
    e na tabela de ausência/unlimited.
- SCP-010.1 afirmava que a SCP-010 havia sido "reescrita
  integralmente", mas o documento real ainda continha o DTO com
  valores inválidos e sources inexistentes.

## 3. Correções aplicadas (documentais)

- Substituição **integral** do arquivo
  `docs/architecture/impact-analysis/SCP-010-commercial-seat-limit-runtime-contract-planning.md`
  com um único H1, uma única seção `## Status`, uma única linha

- Remoção completa de `within_limit`, `tenant_entitlement`,

- Atualização da SCP-010.1 removendo a afirmação incorreta sobre

## 4. DTO canônico preservado (SCP-009)

```
type CommercialLimitDecision = {
  tenantId: string;
  featureKey: string;
  allowed: boolean;
  reason:
    | "entitled"
    | "not_entitled"
    | "limit_reached"
    | "billing_unknown"
    | "billing_attention_required"
    | "billing_blocked"
    | "not_evaluated";
  source: "tenant" | "plan" | "default" | "none";
  limit: number | null;
  used: number | null;
  requestedIncrement: number;
  remaining: number | null;
};
```

Valores proibidos e removidos: `within_limit`, `tenant_entitlement`,
`plan_entitlement`, `override`, `system`, `unknown`,
`invalid_limit`.

## 5. Relação com `CommercialFeatureDecision`

Ordem canônica preservada:
`requireTenant → membership auth → normalizeFeatureKey →
evaluateFeatureCatalogGate → getCommercialFeatureDecision →
(somente se allowed=true e reason="entitled") resolver limite → ler
uso → emitir CommercialLimitDecision`.

Feature decision negativa **propaga** `reason`/`source` sem
reinterpretação (§9.1 do documento SCP-010 consolidado). Não há
colapso automático para `not_evaluated`.

## 6. Resolução do risco de dual-path

Documento SCP-010 §10 fixa a **Alternativa A** — reuso do resolver
comercial autoritativo (`getCommercialFeatureDecision` /
`getTenantEntitlementSnapshot`) para extração do valor numérico —
proibindo a SCP-011 de reimplementar precedência
`tenant → plan → default → none` com queries próprias. Se a extração
exigir refatoração, um resolver compartilhado server-only
(Alternativa B) será proposto como pré-requisito antes de qualquer
runtime novo.

## 7. Mapeamento persistido → DTO

- `tenant_entitlements.source = 'override'` → `"tenant"`.
- `tenant_entitlements.source = 'plan'` (herdada) e resolução via
  `tenant_subscriptions → commercial_plan_entitlements` → `"plan"`.
- `tenant_entitlements.source = 'system'` ou default explícito →
  `"default"`.
- Ausência ou resolução inválida → `"none"`.

Evidência da coluna persistida: migration
`supabase/migrations/20260708223211_*.sql` com CHECK
`source IN ('plan','override','system')`.

## 8. Mapeamento de billing

Nenhuma reinterpretação isolada dos status `trialing | active |
past_due | suspended | canceled | internal | demo`. As razões
comerciais emitidas por `getCommercialFeatureDecision` /
`decideCommercialFeature` / `getTenantBillingHealth` (arquivos:
`src/lib/api/commercial/feature-gate.ts`,
`src/lib/api/commercial/commercial.functions.ts`,
`src/lib/api/commercial/read-models.ts`) são propagadas
literalmente para o DTO `CommercialLimitDecision`.
