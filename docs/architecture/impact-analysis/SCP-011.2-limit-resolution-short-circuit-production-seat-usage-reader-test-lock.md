Accepted

## 1. Objetivo

Fechar os dois bloqueios remanescentes da cadeia SCP-011 / SCP-011.1:

1. Impedir qualquer leitura de `public.tenant_members` quando o limite
   comercial `users.seats` estiver **ausente, não efetivo ou inválido**.
2. Extrair o reader real de produção para um módulo próprio
   (`seat-usage-reader.ts`) e garantir que runtime e specs importem
   exatamente a mesma implementação — sem reader paralelo na suíte.

## 3. Short-circuit do limite (`resolveCommercialSeatLimitDecision`)

Ordem canônica preservada:

```
requireTenant
→ strict input validation
→ featureKey fixada em users.seats (server-side)
→ evaluateFeatureCatalogGate
→ (se catalog liberar)
     loadCommercialContext
     decideCommercialFeature
→ (se feature liberar)
     extractSeatLimit
→ short-circuit: se extracted.limit === null → not_evaluated / none
                 sem chamar readSeatUsage
→ readSeatUsage (uma única vez)
→ decideCommercialSeatLimit
```

Resultado do short-circuit:

```
allowed = false
reason = not_evaluated
source = none
limit = null
used = null
remaining = null
requestedIncrement = 1
```

`limit = 0` é um inteiro comercial válido e **não** dispara short-circuit —
o reader é chamado exatamente uma vez e a decisão emitida é
`limit_reached` (com `remaining = 0`).

## 4. Reader de produção (`readCommercialSeatUsage`)

Assinatura:

```ts
readCommercialSeatUsage(
  client: CommercialSeatUsageClient,
  tenantId: string,
): Promise<number | null>
```

Query executada, exatamente uma vez:

```ts
client
  .from("tenant_members")
  .select("*", { count: "exact", head: true })
  .eq("tenant_id", tenantId)
  .in("membership_status", ["active", "invited"]);
```

Regras:

- `tenantId` obrigatório, string não vazia — string inválida ⇒ retorna
  `null` sem executar query;
- COUNT exato, `head=true`, nenhuma linha retornada;
- `active` e `invited` contam; `suspended` e `revoked` são ignorados;
- roles não alteram a contagem;
- retorna `null` em erro do banco, `count` ausente/negativo/fracionário/
  não finito ou acima de `Number.MAX_SAFE_INTEGER`.

A interface `CommercialSeatUsageClient` expõe apenas as capacidades
necessárias à query — nenhum outro método do supabase-js é alcançável
através dela.

## 5. Uso no runtime público

`getCommercialSeatLimitDecision` deixou de conter a query inline. O
adapter injetado é:

```ts
readSeatUsage: async (tid) => {
  const admin = await getAdmin();
  return readCommercialSeatUsage(admin, tid);
}
```

Com `getAdmin` fazendo carregamento lazy compartilhado. Assim:

- catalog gate negativo ⇒ `loadAdmin` não é chamado;
- feature decision negativa ⇒ reader não é chamado;
- feature positiva + limite ausente/inválido ⇒ reader não é chamado;
- feature positiva + limite válido ⇒ reader chamado uma única vez.
