# GA-08.0 — Documentation Repository Structure Audit & Controlled Migration Plan

## Status

Accepted

## 1. Objetivo

Executar auditoria read-only completa do repositório documental do
RM Prime SaaS, quantificar a documentação existente, classificar os
128 arquivos legados residentes em `docs/fase6/` por macrofase real
registrada no `ROADMAP_ARCHITECTURAL.md` e propor um plano de migração
controlada para uma estrutura canônica de longo prazo.

Esta etapa é exclusivamente de análise. Nenhum arquivo foi criado,
movido, renomeado ou alterado durante a GA-08.0. A movimentação física
é executada pela subetapa subsequente **GA-08.1**.

## 2. Escopo

- Auditoria estrutural de `docs/**`.
- Contagem determinística de arquivos por diretório.
- Classificação dos 128 arquivos legados em `docs/fase6/` conforme a
  macrofase real em que foram produzidos.
- Definição da estrutura canônica de destino.
- Definição do mapeamento antigo → novo.
- Registro de aprovação externa da classificação.

## 3. Repositório auditado

- Repositório: `MrRodBH/prime-domus-hub`.
- Branch: `main`.
- Estado inicial imediatamente antes da GA-08.1.

## 4. Contagens confirmadas

```text
docs/architecture = 47 arquivos
docs/fase6        = 128 arquivos
total             = 175 arquivos
```

## 5. Causa-raiz da reorganização

O diretório `docs/fase6/` foi utilizado como repositório cronológico
provisório de relatórios ao longo das Fases 2, 3 e 4 e do refactor
inicial de UX. Isso produziu três efeitos indesejáveis:

- perda da correspondência 1:1 entre relatório e macrofase;
- crescimento monotônico do diretório sem segmentação por iniciativa;
- ausência de contrato documental normativo sobre onde novos
  relatórios devem residir.

A GA-08 institui um contrato documental permanente que corrige esses
três efeitos.

## 6. Classificação dos 128 arquivos legados

A classificação é derivada dos prefixos numéricos de criação
histórica dos relatórios, correspondendo à macrofase real em que a
etapa foi executada.

```text
prefixos 00–09  → 29 arquivos → Product UX Refactor
prefixos 10–27  → 18 arquivos → Fase 2 — Multi-Tenant Core
prefixos 28–42  → 15 arquivos → Fase 3 — Membership Evolution Model
prefixos 43–108 → 66 arquivos → Fase 4 — SaaS Commercial Platform
```

Totais: `29 + 18 + 15 + 66 = 128`.

Esta classificação foi validada externamente e é o insumo autoritativo
para a GA-08.1. Não é reinterpretada com base no nome do diretório
atual.

## 7. Estrutura escolhida

A estrutura canônica de destino é definida pelo Document Repository
Contract (`docs/governance/DOCUMENT_REPOSITORY_CONTRACT.md`) e resumida
abaixo:

```text
docs/architecture/                (preservado)
docs/delivery/product-roadmap/phase-06-product-ux-refactor/
docs/delivery/architectural-roadmap/phase-02-multi-tenancy/
docs/delivery/architectural-roadmap/phase-03-membership-evolution/
docs/delivery/architectural-roadmap/phase-04-saas-commercial-platform/
docs/governance/
```

`docs/fase6/` é classificado como caminho legado, será removido pela
GA-08.1 e é proibido para novos documentos.

## 8. Mapeamento antigo → novo

O mapeamento é determinístico e derivado do prefixo numérico do nome
do arquivo:

```text
00 ≤ prefixo ≤ 09 → docs/delivery/product-roadmap/phase-06-product-ux-refactor/
10 ≤ prefixo ≤ 27 → docs/delivery/architectural-roadmap/phase-02-multi-tenancy/
28 ≤ prefixo ≤ 42 → docs/delivery/architectural-roadmap/phase-03-membership-evolution/
43 ≤ prefixo ≤ 108 → docs/delivery/architectural-roadmap/phase-04-saas-commercial-platform/
```

Exemplos representativos:

```text
docs/fase6/00-principios-arquiteturais.md
  → docs/delivery/product-roadmap/phase-06-product-ux-refactor/00-principios-arquiteturais.md

docs/fase6/27-phase-2-closing-review.md
  → docs/delivery/architectural-roadmap/phase-02-multi-tenancy/27-phase-2-closing-review.md

docs/fase6/42-f3-7-phase-3-closing-review.md
  → docs/delivery/architectural-roadmap/phase-03-membership-evolution/42-f3-7-phase-3-closing-review.md

docs/fase6/108-scp-012-0-1-1-deterministic-full-section-rewrite-evidence-lock-git-readiness.md
  → docs/delivery/architectural-roadmap/phase-04-saas-commercial-platform/108-scp-012-0-1-1-deterministic-full-section-rewrite-evidence-lock-git-readiness.md
```

## 9. Aprovação externa

A classificação `29/18/15/66` e o mapeamento por prefixo foram
aprovados pela auditoria externa como insumo autoritativo da GA-08.1.

- nenhum conteúdo documental foi alterado;
- nenhum código, teste, migration, schema, RLS, grant, provider ou

- `docs/fase6/` permaneceu íntegro para consumo da GA-08.1.
